Hexo安装和使用

1. 环境安装

1.1 安装 homebrew

$ /bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/master/install.sh)"

1.2 安装node.js环境和npm(如果已经安装自行跳过)

$ brew install node

1.3 因为众所周知的原因，可选安装 cnpm(已经安装的自行跳过)

$ npm install -g cnpm --registry=https://registry.npm.taobao.org

2. 安装Hexo

$ cnpm install -g hexo

3. Hexo使用

3.1 在电脑中建立一个名字叫「Hexo」的文件夹

$ hexo init

Hexo随后会自动在目标文件夹建立网站所需要的文件. 然后:

$ cnpm install

会在项目根目录中安装 node_modules。

3.2. 启动

运行下面的命令

$ hexo server

表明Hexo Server已经启动了，在浏览器中打开 http://localhost:4000/, 这时可以看到Hexo已为你生成了一篇blog。你可以按 Ctrl+C 停止Server。

3.3 创建新的文章

$ hexo new "My New Post"

ctrl C停止server

$ hexo server

刷新 http://localhost:4000/，可以发现已生成了一篇新文章 “My New Post”。

3.4 编辑

可以使用markdown语法编辑文章

4. 将 Hexo 部署到 GitHub Pages

** 以下内容来自于Hexo部署文档, 文档内容有部分不适用当前Git环境，做了部分修改 **

我们将会使用 Travis CI 将 Hexo 博客部署到 GitHub Pages 上。Travis CI 对于开源 repository 是免费的，但是这意味着你的站点文件将会是公开的。如果你希望你的站点文件不被公开，请直接前往本文 Private repository 部分。

4.1 新建一个 repository。如果你希望你的站点能通过 <你的 GitHub 用户名>.github.io 域名访问，你的 repository 应该直接命名为 <你的 GitHub 用户名>.github.io。
4.2 将你的 Hexo 站点文件夹推送到 repository 中。默认情况下不应该 public 目录将不会被推送到 repository 中，你应该检查 .gitignore 文件中是否包含 public 一行，如果没有请加上。
4.3 将 Travis CI 添加到你的 GitHub 账户中。
4.4 前往 GitHub 的 Applications settings，配置 Travis CI 权限，使其能够访问你的 repository。
4.5 你应该会被重定向到 Travis CI 的页面。如果没有，请 手动前往。
4.6 在浏览器新建一个标签页，前往 GitHub 新建 Personal Access Token，只勾选 repo 的权限并生成一个新的 Token。Token 生成后请复制并保存好。
4.7 回到 Travis CI，前往你的 repository 的设置页面，在 Environment Variables 下新建一个环境变量，Name 为 GH_TOKEN，Value 为刚才你在 GitHub 生成的 Token。确保 DISPLAY VALUE IN BUILD LOG 保持 不被勾选 避免你的 Token 泄漏。点击 Add 保存。
4.8 在你的 Hexo 站点文件夹中新建一个 .travis.yml 文件：
$ touch .travis.yml

内容:

sudo: false
language: node_js
node_js:
    - 10 # use nodejs v10 LTS
cache: npm
branches:
    only:
        - master # build master branch only
script:
    - hexo generate # generate static files
deploy:
    provider: pages
    skip-cleanup: true
    github-token: $GH_TOKEN
    keep-history: true
    on:
        branch: master
    local-dir: public
4.9 将 .travis.yml 推送到 repository 中。Travis CI 应该会自动开始运行，并将生成的文件推送到同一 repository 下的 gh-pages 分支下
4.10 前往 https://<你的 GitHub 用户名>.github.io 查看你的站点是否可以访问。这可能需要一些时间。
